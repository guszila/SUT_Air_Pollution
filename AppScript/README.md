##  LINE Official Account & Chatbot Integration

ระบบแจ้งเตือนและบริการข้อมูลคุณภาพอากาศผ่านแอปพลิเคชัน LINE (LINE OA) เพื่อให้กลุ่มผู้ใช้งาน (นักศึกษาและบุคลากร มทส.) สามารถเข้าถึงข้อมูลสภาพอากาศในพื้นที่ได้แบบ Real-time และง่ายดายที่สุด

>  Code Google AppScript : `SAQ_AppScript.js`

> [!IMPORTANT]
**ข้อควรระวังเรื่องความปลอดภัย:**
โค้ดไฟล์ `SAQ_AppScript.js` อย่าลืมกรอก Line Token `"YOUR_LINE_CHANNEL_ACCESS_TOKEN"` และไม่ควรแบ่งปัน Line Token ให้ใคร

###  ฟีเจอร์หลัก (Key Features)

* **Automated Greeting Workflow:** เมื่อมีผู้ใช้งานเพิ่มเพื่อนใหม่ ระบบจะส่งข้อความต้อนรับอัตโนมัติ (Greeting Message) พร้อมแนบลิงก์แบบประเมินความพึงพอใจทันที เพื่อสร้างความร่วมมือและเก็บฟีดแบ็คตั้งแต่เริ่มใช้งาน
* **Intuitive Rich Menu:** ออกแบบปุ่มสั่งการขนาดใหญ่ด้านล่างหน้าจอแชท แยกเมนูการใช้งานชัดเจน ช่วยให้ผู้ใช้กดเช็คค่าฝุ่นปัจจุบันได้ทันทีโดยไม่ต้องพิมพ์คำสั่ง
* **Real-time AQI Alerts:** ระบบพร้อมรองรับการแจ้งเตือนเชิงรุก (Push Notification) เมื่อค่าฝุ่น PM2.5 เกินมาตรฐาน เพื่อให้ผู้ใช้เตรียมตัวสวมหน้ากากอนามัยหรือเลี่ยงกิจกรรมกลางแจ้ง
* **AI Chatbot Integration (Beta):** อยู่ในขั้นตอนการพัฒนาและทดสอบระบบตอบกลับอัจฉริยะ (Natural Language Processing) เพื่อให้บอทสามารถตอบคำถามทั่วไปเกี่ยวกับสุขภาพและพิกัดจุดติดตั้งเครื่องได้อัตโนมัติ
* **Nearby Station Tracking (เช็คค่าฝุ่นที่ใกล้ที่สุดในระยะ 3 กม.):** ผู้ใช้งานสามารถกดแชร์ตำแหน่งที่ตั้ง (Location) ของตัวเองเข้ามาในไลน์ ระบบหลังบ้านจะคำนวณระยะทางระหว่างตัวผู้ใช้กับเครื่องตรวจวัดทุกเครื่องใน มทส. ทันที 
  * **กรณีที่อยู่ใกล้เครื่องตรวจวัด (ในระยะ 3 กม.):** ไลน์จะดึงค่าฝุ่นจากเครื่องที่อยู่ใกล้ที่สุดส่งกลับไปให้ทันที
  * **กรณีที่อยู่ไกลเกินไป (เกินระยะ 3 กม.):** ไลน์จะแจ้งเตือนผู้ใช้ทันทีว่า "ไม่อยู่ในระยะบริการของเครื่องตรวจวัด" เพื่อป้องกันความคลาดเคลื่อนของข้อมูล

> [!TIP]
> **สูตรการคำนวณระยะทางที่ใช้เบื้องหลัง:**
> ในโค้ด `LineWebhook.js` จะใช้สูตรคำนวณระยะทางระหว่างพิกัดสองจุด (ละติจูด/ลองจิจูด) เพื่อหาค่าห่างเป็นกิโลเมตร ถ้าผลลัพธ์ค่าน้อยกว่าหรือเท่ากับ `3.0` ระบบถึงจะดึงข้อมูลฝุ่นมาแสดงผล

###  ขั้นตอนการทำงานของระบบ (User Workflow)

1. **Add Friend** → ผู้ใช้สแกน QR Code เพื่อเพิ่มเพื่อนระบบ `SUT Air Pollution`
2. **Onboarding Survey** → ระบบส่งข้อความต้อนรับพร้อมลิงก์ Google Form ทันที เพื่อเก็บข้อมูลความพึงพอใจกลุ่มเป้าหมาย
3. **Menu Interaction** → ผู้ใช้กดปุ่มบน **Rich Menu** เพื่อเรียกดูค่าฝุ่น (PM1.0, PM2.5, PM10) และสภาพอากาศ (อุณหภูมิ/ความชื้น) ปัจจุบันที่ดึงมาจากคลาวด์ Google Sheets

###  การตั้งค่าและสถาปัตยกรรมระบบ (Backend Architecture)

* **LINE Messaging API & Webhooks:** ใช้สำหรับรับส่งข้อมูลคำสั่งระหว่างแอปพลิเคชัน LINE และเซิร์ฟเวอร์
* **Google Apps Script (GAS):** ทำหน้าที่เป็นตัวกลาง (Webhook Receiver) คอยดึงค่าล่าสุดจากฐานข้อมูล Google Sheets ส่งกลับไปแสดงผลบน LINE หน้าจอแชทของผู้ใช้

### ขั้นตอนการติดตั้งและการตั้งค่าระบบ (Installation & Setup)

#### 1. การตั้งค่าฝั่ง LINE Developers
1. สมัครบัญชีผู้ใช้งานและสร้าง Provider ใหม่ใน [LINE Developers Console](https://developers.line.biz/)
2. สร้าง Channel ใหม่โดยเลือกประเภทเป็น **Messaging API**
3. กรอกข้อมูลพื้นฐานของโปรเจกต์ให้ครบถ้วน จากนั้นให้คัดลอกค่า **Channel Access Token** และ **Channel Secret** เก็บไว้สำหรับใช้เชื่อมต่อระบบหลังบ้าน

#### 2. การเชื่อมต่อระบบ Webhook (Google Apps Script)
1. นำโค้ด JavaScript หลังบ้าน (Google Apps Script) ที่เตรียมไว้ไปวางในตัวแก้ไขสคริปต์ของ Google Sheets
2. นำค่า **Channel Access Token** จาก LINE Developers มาใส่ในตัวแปรที่กำหนดไว้ในโค้ดเพื่อระบุปลายทางในการส่งข้อความกลับ (Reply Message)
3. ทำการกด **Deploy (การทำงานใช้จริง)** เลือกประเภทเป็น **Web App (แอปพลิเคชันเว็บ)** และตั้งค่าให้ "ทุกคน (Anyone)" สามารถเข้าถึงได้
4. คัดลอก URL ของ Web App ที่ระบบสร้างให้ (เช่น `https://script.google.com/.../exec`)
5. นำ URL ดังกล่าวกลับไปกรอกในช่อง **Webhook URL** บน LINE Developers Console พร้อมกับกดเปิดใช้งานฟังก์ชัน **Use Webhook** และปิดฟังก์ชันข้อความตอบกลับเริ่มต้นของ LINE

#### 3. การตั้งค่าข้อความต้อนรับและ Rich Menu (LINE OA Manager)
1. เข้าสู่ระบบจัดการที่ [LINE Official Account Manager](https://manager.line.biz/)
2. **ตั้งค่าข้อความทักทายเพื่อนใหม่ (Greeting Message):** เปิดการใช้งานและกรอกข้อความต้อนรับ พร้อมสอดแทรกลิงก์ Google Form สำหรับทำแบบประเมินความพึงพอใจการใช้งาน
3. **ตั้งค่าเมนูหลัก (Rich Menu):** ทำการอัปโหลดรูปภาพหน้าปกเมนูที่ออกแบบไว้ และกำหนด Action ของแต่ละปุ่มให้ส่งค่า Text (คีย์เวิร์ด) หรือลิงก์ไปยัง Web Application เพื่อให้ระบบหลังบ้านดึงข้อมูลค่าฝุ่นมาแสดงผลตามคำสั่ง
